
logger.py设计思路：
    将日志分为两类，职责分离：
    ┌─────────────────────────────────────────────────────┐
    │  运维日志 (Ops Log)  │  记录系统健康状态与性能        │
    │                      │  仅在控制台输出，不写文件      │
    ├─────────────────────────────────────────────────────┤
    │  业务日志 (Event Log)│  记录状态机事件与报警变化      │
    │                      │  同时输出控制台 + 写入文件     │
    └─────────────────────────────────────────────────────┘

日志级别（运维日志专用）：
    DEBUG    → 每帧细节，如检测框坐标、计数器值（开发调试用，上线应关闭）
    INFO     → 正常流程节点，如模型加载成功、心跳发送
    WARNING  → 性能下降或边缘情况，如 FPS 过低
    ERROR    → 可恢复的错误，如 UART 单次发送失败（系统继续运行）
    CRITICAL → 不可恢复的致命错误，如模型加载失败（系统将退出）

Event 命名规则（业务日志专用）：
    格式：领域.动作.结果
    示例：alarm.state.on / uart.send.fail / system.boot.ok
    规范：
        - 全小写，用英文句点分隔
        - 动词使用现在时（detect / send / load）
        - 结果词只使用：ok / fail / enter / exit / timeout / on / off
