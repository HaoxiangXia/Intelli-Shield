# 叉车安全仿真演示系统说明

## 概述

仿真演示系统用于在不接入真实设备、不影响正式数据库的情况下，展示首页地图上的叉车运行与安全告警效果。

演示入口为：

```text
cloud/tests/edge_node_client.py
```

启动后系统会自动运行 FastAPI 服务、创建独立演示数据库，并在首页地图中显示 3 台叉车。叉车沿固定路线匀速移动，演示过程中会按剧本触发碰撞预警、行人预警、行人报警和报警解除。

## 运行方式

在项目的 `cloud` 目录下执行：

```powershell
.\.venv\Scripts\python.exe tests\edge_node_client.py
```

脚本启动成功后会在终端输出访问地址，例如：

```text
仿真演示系统已启动，数据库: C:\...\cloud\alarm_demo.db
http://localhost:51247
```

浏览器会自动打开首页。如果没有自动打开，可手动访问终端输出的 URL。

## 数据库隔离

演示系统使用独立数据库文件：

```text
cloud/alarm_demo.db
```

同时可能产生 SQLite WAL 相关文件：

```text
cloud/alarm_demo.db-wal
cloud/alarm_demo.db-shm
```

每次启动演示脚本时都会重建 `alarm_demo.db`。正式数据库 `cloud/alarm.db` 不会被删除、覆盖或恢复，演示数据与正式数据保持隔离。

## 首页地图效果

首页地图展示 3 台叉车：

| 设备 ID | 说明 |
|---------|------|
| `FORK-001` | 沿左侧矩形路线运行 |
| `FORK-002` | 沿下方矩形路线运行，并参与行人预警/报警演示 |
| `FORK-003` | 沿右侧矩形路线运行，并参与交叉点碰撞预警演示 |

地图上的叉车标记使用较小圆点显示。每台叉车周围有两层同心虚线圆：

| 圆环 | 半径 | 含义 |
|------|------|------|
| 外圈 | `90` 地图坐标单位 | 预警范围 |
| 内圈 | `45` 地图坐标单位 | 报警范围 |

叉车状态颜色由设备状态决定：

| 状态 | 表现 |
|------|------|
| 正常在线 | 绿色 |
| 报警中 | 红色 |
| 离线 | 灰紫色 |

## 固定路线

地图坐标采用 `1920 x 1080` 坐标系。三台叉车路线如下：

```python
FORK-001 = [(630, 760), (630, 150), (150, 150), (150, 760), (630, 760)]
FORK-002 = [(630, 760), (1417, 760), (1417, 950), (630, 950), (630, 760)]
FORK-003 = [(630, 760), (1417, 760), (1417, 150), (630, 150), (630, 760)]
```

叉车按路径线段插值移动，当前演示速度为：

| 设备 ID | 速度 |
|---------|------|
| `FORK-001` | `36` 地图坐标单位/秒 |
| `FORK-002` | `42` 地图坐标单位/秒 |
| `FORK-003` | `48` 地图坐标单位/秒 |

## 演示剧本

演示按循环剧本运行：

| 阶段 | 时长 | 行为 |
|------|------|------|
| 正常巡航 | 15 秒 | 3 台叉车沿固定路线匀速行驶 |
| 交叉点预警 | 10 秒 | `FORK-001` 和 `FORK-003` 接近 `[630,760]`，触发碰撞预警 |
| 行人预警 | 8 秒 | 行人进入 `FORK-002` 安全预警区，`FORK-002` 进入报警状态 |
| 行人报警 | 10 秒 | 行人进一步进入报警区，`FORK-002` 保持报警状态 |
| 报警解除 | 10 秒 | 行人远离，报警解除，叉车继续运行 |

行人位置由仿真内部维护，不会作为第 4 个地图点展示。首页地图始终只显示 3 台叉车。

## 实时刷新机制

仿真线程会周期性写入 `alarm_demo.db` 中的设备位置和报警状态，并通过 Socket.IO 发送 `device_update` 事件。

当前首页 `Dashboard.vue` 会在收到 `device_update` 后重新请求：

```text
GET /api/devices
GET /api/recent-alarms?limit=10
```

因此地图位置、设备状态和告警列表会随仿真循环自动刷新。

## 前端构建

如果修改了首页地图显示逻辑，需要重新构建前端：

```powershell
cd cloud\frontend
npm run build
```

后端服务的是 `cloud/frontend/dist` 中的构建产物。只修改 `src` 文件但不构建时，演示脚本打开的页面不会显示最新前端效果。

## 常见问题

### 页面没有显示最新圆环或标记大小

确认是否已经执行：

```powershell
cd cloud\frontend
npm run build
```

然后重新运行演示脚本。

### 设备突然离线

演示脚本会将离线超时调整为较长时间，并持续刷新 `last_seen`。如果出现离线，通常是脚本异常退出或浏览器连接到了其他后端实例。

### 正式数据会不会受影响

不会。演示脚本会把后端运行时数据库路径切换到 `cloud/alarm_demo.db`，不会操作 `cloud/alarm.db`。

### 如何调整移动速度

修改 `cloud/tests/edge_node_client.py` 中的：

```python
FORKLIFT_SPEEDS = {
    "FORK-001": 36.0,
    "FORK-002": 42.0,
    "FORK-003": 48.0,
}
```

数值越小，地图上的叉车移动越慢。

### 如何调整预警和报警范围

修改 `cloud/frontend/src/views/Dashboard.vue` 中的：

```js
const MAP_WARNING_RADIUS = 90
const MAP_DANGER_RADIUS = 45
```

修改后需要重新执行 `npm run build`。
